### Problema 

La organización necesita obtener y analizar datos provenientes de una fuente externa, pero actualmente no cuenta con un sistema que automatice este proceso. La recolección de información debe hacerse manualmente a través de una API, lo que provoca demoras, inconsistencias y una alta probabilidad de error. Además, los datos obtenidos no se almacenan de forma estructurada, por lo que su procesamiento para análisis estadístico y probabilístico se vuelve lento y poco eficiente. Tampoco existen mecanismos que permitan ejecutar tareas periódicas para mantener actualizado el conjunto de datos, lo cual afecta la oportunidad y la calidad del análisis.

Ante esta situación, se vuelve necesario diseñar y construir una canalización de datos capaz de obtener la información automáticamente, almacenarla en una base de datos mediante herramientas adecuadas y procesarla con métodos estadísticos y probabilísticos usando Python. Esta canalización debe permitir también la creación de gráficos y visualizaciones que faciliten la interpretación de resultados, además de estar acompañada por documentación clara del sistema desarrollado. El problema de fondo es la ausencia de un proceso integrado que transforme datos crudos en información confiable y útil para el análisis, de manera eficiente, automatizada y sostenible en el tiempo.
